# DBRepo Load Verification

## Purpose

This document records the C-role verification of the DBRepo loading and view-access workflow for the project **Forecasting Demographic Changes in the Population in Vienna using Machine Learning**.

## DBRepo implementation clarification

According to the D-role/API implementation, the current DBRepo workflow is based on the normalized DBRepo schema using the raw input dataset structure:

`vie-bez-pop-sex-age5-stk-nat-geo4-2002f.csv`

The processed long-format dataset:

`processed_population_long_v1.csv`

is a derived ML-ready artefact generated by `src/01_preprocess_data.py`. It is useful for local preprocessing and modelling, but it is not the primary dataset structure used for the normalized DBRepo implementation.

## Repository files checked

The repository contains the raw input dataset:

`data/vie-bez-pop-sex-age5-stk-nat-geo4-2002f.csv`

The repository also contains the processed long-format dataset:

`data/processed_population_long_v1.csv`

The processed dataset is generated by:

`src/01_preprocess_data.py`
