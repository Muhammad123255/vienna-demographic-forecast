# DBRepo Load Verification

## Purpose

This document records the C-role verification of the DBRepo loading and view-access workflow for the project **Forecasting Demographic Changes in the Population in Vienna using Machine Learning**.

## Repository files checked

The repository contains the raw input dataset:

`data/vie-bez-pop-sex-age5-stk-nat-geo4-2002f.csv`

The repository also contains the processed long-format dataset:

`data/processed_population_long_v1.csv`

The processed dataset is generated by:

`src/01_preprocess_data.py`

## Issue found

The repository contains two different API access approaches.

The notebook `notebooks/dbrepo_api_test.ipynb` tries to access view data using URLs like:

`https://test.dbrepo.tuwien.ac.at/api/dataset/38707917-e942-45c3-a3dd-d2bfc1c106af/view/population_by_district`

The saved notebook output shows HTTP `404` responses for these view endpoints.

The script `src/dbrepo_api_loader.py` also needs alignment because it currently uses:

`https://dbrepo.tuwien.ac.at/api`

and tests view names that do not match `sql/views.sql`.

## Required fix before final sign-off

The DBRepo API workflow should be updated to use the test DBRepo endpoint and the actual created view IDs.

Recommended approach:

- use the DBRepo Python client endpoint `https://test.dbrepo.tuwien.ac.at`
- use database ID `38707917-e942-45c3-a3dd-d2bfc1c106af`
- retrieve view data by view ID using:
  - `1448ed5a-3777-482c-8857-ea83a590f416`
  - `9517ff56-7559-4999-88db-0236bcffcdd4`
  - `40d0e45f-3e41-4c0d-8c2e-41d8967be4c2`
